TemLang

TemLang is a programming language designed to make creating C programs easier. TemLang compiles to C code that can be included into C projects. This language was not designed to create aplications on its own. It was designed to be used in conjunction with C.

Value Types

  • Null

    'Null' is the only valid value for a null type.

    Example

    let x null // Create variable 'x' as a null value

  • Boolean

    True or False

    Example

    let x true
    let y false

  • Number

    Numbers can either a floating-point number or an integer. Numbers can also have a range assigned to them. A number with a range cannot be assigned a number outside of its range. The range of a number also cannot be changed once assigned.

    Ranges

    • u8 0 → 255
    • u16 0 → 65,535
    • u32 0 → 4,294,967,295
    • u64 0 → 18,446,744,073,709,551,615
    • i8 -128 → 127
    • i16 -32,768 → 32,767
    • i32 -2,147,483,648 → 2,147,483,647
    • i64 -9,223,372,036,854,775,808 → 9,223,372,036,854,775,807
    • f32 -3.40282347E+38 → 3.40282347E+38
    • f64 -1.7976931348623157e+308 → 1.7976931348623157e+308

    Example

    mutableLet x 10 // Unranged number with the value 10
    set x -1 // This is okay because the number has no range

    mutableLet y 10 + u8 // Number with the range of [0, 255]
    set y -1 // ERROR: this will give an error because -1 is not in the range

  • String

    String are a series of ASCII characters

    Example

    mutableLet message "Hello World" // Declare a variable 'message' as a string
    append message '!' // Append a '!' character. Message is now "Hello World!"
    pop message // Pop the last character from the list. Message is now "Hello World"

  • Enumeration

    A set of possible values where only one value is valid at an instance.

    Example

    enum color { red green blue }
    mutableLet c "red" + #color // Declare an enumeration instance
    set c "green" + #color // Set enumeration instance to new value

  • Flag

    A set of possible values where none to all of values are valid at an instance.

    Example

    flag computerState { sleeping overHeated lowMemory }
    mutableLet cs null + #computerState // Declare an flag instance with all flags set to off
    on cs sleeping // Set the sleeping flag to on
    off cs sleeping // Set the sleeping flag to off
    all cs // Set all the flags to on
    clear cs // Set all the flags to off

  • Data Structure

    A collection of value types grouped together to make one value type.

    Example

    let anonymous 
{{ 
    let x 10 
    let y  "This is a message" 
    let z [ 0 2 3 4 5 ] 
}}
// This is an anonymous data structure. '{{ }}' is used for struct initialization.

struct vector3
{ 
    f32 x 
    f32 y 
    f32 z 
}
let v3 
{{ 
    let x 10 
    let y 20 
    let z 30 
}} + #vector3
// This defines a structure and type checks it to see if it matches 'vector3'

  • Data Variant

    A collection of value types where only one value type is valid at an instance.

    Example

    variant Pet
{ 
    #Dog dog
    #Cat cat
    #Bird bird
}
let pet 
{{ 
    let dog {{  /*Dog initialization goes here...*/  }} + #Dog
}} + #Pet
// This defines a variant 'Pet' that is set to 'dog' as its current value.

variant ExampleVariant
{ 
    i32 number1
    i32 number2
}
let example 
{{ 
    let number2 42 + i32 
}} + #ExampleVariant
// This defines a variant 'ExampleVariant' that is set to 'number2' as its current value. Notice that the name of the variable in the struct indicates what value the variant will have.

  • List/Array

    A collection of value types. Lists can increase/decrease the number of values in the collection. Arrays have a fixed length.

    Example

    let numberArray [| (0 + i32) 1 2 3 4 |] // '[| |]' is for array initialization.
    mutableLet numberList [ (0 + i32) 1 2 3 4 ] // '[ ]' is for list initialization.
    append numberList 5 // [ 0 1 2 3 4 5 ]
    pop numberList // [ 0 1 2 3 4 ]

  • Type

    A type definition to be added to another value as a way to perform type checking or casting.

    Built-in Types

    • null
    • bool
    • u8
    • u16
    • u32
    • u64
    • i8
    • i16
    • i32
    • i64
    • f32
    • f64
    • string
    • external

    Example

    let x 0 + i32 // Define a 32 bit integer
    let y x + f32 // Use the value of x but convert it to a floating-point number
    let z y + string // Convert the value of y to a string

Expressions

Expressions are a series of operators and operands that evaluate to a value.

Operands

  • Nullary See null
  • Unary Value See value types
  • Unary Variable
  • Unary Scope A series of instructions that return a value

    Example

    let result 
{
    let x 10
    let y 20
    return x + y
}
    // Result will be assigned the value 30. Variables in a scope cannot be accessed outside of a scope.
  • Unary Struct See struct
  • Unary List See list
  • Unary Match Selects a expression to evaluate based on the target expression

    Example

    let result
{= 
    x // Target expression is 'x'.
    1 x // If 'x' is 1, expression evaluates to 'x'.
    2 (x * 2) // If 'x' is 2, expression evaluates to 'x * 2'.
    3 { return x + 42 } // If 'x' is 3, expression evaluates to 'x + 42'.
    x // Default case. If previous values aren't matched, then expression evaluates to 'x'.
=}
  • Binary Two operands joined together by an operator to make a value
    Left Operand Operator Right Operand Comments
    Number + Number
    String + String Concatenate strings
    String + Type (enum or flag) Commutative, Make enum with member
    Struct + String Commutative, Type check struct or create variant
    Number - Number
    Null - Number Commutative, Negate number
    Number * Number
    Number / Number
    Number % Number
    Boolean | Boolean
    Boolean ^ Boolean
    Boolean & Boolean
    Null ! Boolean Commutative
    Any Value =, >, < Value of same type
    Null # Type Commutative
    List @ Number Get member of list/array
    Struct @ String Get member of struct
    Null # Name of custom value type (struct, enum, flag, etc.) Commutative, Make type value
    Null ## List/Array Commutative, Get number of values in container
    Null ## Enum/Flag Commutative, Get number of members
    Null [] Type Commutative, Make empty list
    Flag + Flag Activate flag member
    Flag - Flag Deactivate flag member

Function Calls

Functions can be called with just one argument. A null value will be placed where the missing argument is.

arg :func // This parses to arg :func null
:func arg // This parses to null :func arg
- 4 // This parses to null - 4
4 - // This parses to 4 - null

Negative Numbers

All numbers are parsed without accounting for a sign. This means -1 isn't parsed as the number '-1'. It is parsed as null - 1 which evaluates to -1. It is important to understand this so that certain errors can be avoided. For example, 2 + -3 will give an error because it will be parsed as 2 + null - 3 which is invalid. One must write 2 + (-3) to ensure that the 3 is negated before performing any operations.

Capturing variables

All functions, scopes, and procedures can read variables from outer scopes. However, only variables that are captured can be edited within a scope or procedure. Functions cannot capture variables. 

mlet x 42
/*
while (x > 0) ()
{
    /*set x x - 1*/ // x cannot be changed because it wasn't captured in the 'while' instruction
}
*/
while (x > 0) (x) {
    set x x - 1 // x can be changed because it was captured in the 'while' instruction
}
